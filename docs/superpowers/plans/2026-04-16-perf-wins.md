# Performance Wins Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce image payload, tune Next.js + Clerk for smaller client JS and faster FAPI, remove a misleading `<source media>`, add a hero poster, gate the grain overlay to desktop.

**Architecture:** Config- and leaf-level changes. No new abstractions. Every task is independently revertable. `ClerkProvider` placement is **not changed** — root-layout wrapping is Clerk's official pattern.

**Tech Stack:** Next.js 16 App Router, Clerk v7 (Core 3), Tailwind CSS v4, Vitest, sips/ffmpeg for asset manipulation.

### Task 4: Enable Clerk `frontendApiProxy`

**Files:**
- Modify: `src/proxy.ts`

Context: Core 3 feature that proxies Clerk's Frontend API through your own domain. Eliminates a cross-origin DNS+TLS round trip for anyone who has ever been signed in. No env var change.

- [ ] **Step 1: Update proxy.ts**

Replace the body of `src/proxy.ts`:

```ts
import { clerkMiddleware, createRouteMatcher } from "@clerk/nextjs/server";

const isProtectedRoute = createRouteMatcher([
  "/members(.*)",
  "/students(.*)",
  "/student-resources(.*)",
]);

export default clerkMiddleware(
  async (auth, req) => {
    if (isProtectedRoute(req)) {
      await auth.protect();
    }
  },
  { frontendApiProxy: { enabled: true } },
);

export const config = {
  matcher: [
    "/((?!_next/static|_next/image|favicon.ico|images|videos|fonts).*)",
    "/(api|trpc)(.*)",
  ],
};
```

Visit http://localhost:3000/sign-in. In devtools → Network, FAPI requests now go through your own origin (path starts with `/__clerk`).
